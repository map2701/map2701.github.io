Otimizar SQL Server

Esta como problema de desempenho no seu SQL Server ?

Precisando de um DBA Remoto SQL Server entre em contato conosco.

Para Otimizar SQL Server, o primeiro passo é descobrir a causa ou causas,pois podemos ter várias causas como:

  • Pouca memória
  • Disco com velocidade inadequada para servidor de banco de dados
  • Problemas com TempDB
  • Falta de índices e estatísticas
  • Problema  com consultas
  • Processador
  • Excesso de Locks ou DeadLocks

Nos nossos atendimentos a maioria dos problemas estão relacionados com falta de índices e estatísticas, isso ocorre porque a grande maioria das empresas pequenas e médias não possuem um DBA, com isso não é realizado o trabalho de melhoria continua dos ambientes de Banco de Dados.

Por isso indicamos que todas empresas deveriam contar com a consultoria de uma DBA, mesmo que apenas algumas horas por mês, traz  um grande beneficio. A falta desse profissional nas empresas hoje traz grandes perdas para as empresas, pois sem o devido cuidado o desempenho dos ambientes de banco de dados tem um desempenho muito abaixo de sua capacidade.

Todo DBA SQL Server deve possuir scripts e checklists para poder rapidamente otimizar SQL Server e agilizar a solução de problemas. Entre os mais importantes eu citaria script para verificar índices faltando ou fragmentados, atualizar estatísticas, verificação de blocks e locks, verificação de contenção.

Outros itens importantes para serem checados são os contadores do windows para disco, processador e memória mas com cuidado, porque um recurso pode interferir em outro contador. Falta de memória por exemplo sobrecarrega o disco e o processador, com isso se você trocar o disco por um mais rápido ao invés de adicionar mais memória o problema provavelmente não será resolvido e sim apenas amenizado.

Para monitorarmos os locks podemos utilizar esse script do Fabricio Lima, outro script muito importante é o que lista as estatísticas das tabelas.

Otimizar SQL Server com Sp_Blitz o script dos Mestres do SQL Server

Como um DBA SQL Server mais experiente provavelmente você deva utilizar o super script sp_blitz do mestre Brent Ozar e sua equipe. Com esse script você tem uma visão geral do servidor e bancos de dados do seu SQL Server, podendo otimizar SQL Server com uma velocidade bem maior.

Otimizar SQL Server otimizando consultas

Não como Otimizar SQL Server  sem otimizar as consultas, pois consultas ruins utilizam mais recursos do servidor que o necessário, podendo ter casos de somente uma consulta parar todo um ambiente SQL Server.

Existem vários recursos que podemos utilizar para otimizar uma consulta, porém o primeiro passo é gerar o plano de execução da consulta e também utilizar os comandos set statistics io on  e set statistics time on para trazer mais informações sobre a consulta. O plano de execução nos dirá se precisamos criar novos índices já as estatistas de io vão nos mostrando os resultados das modificações, pois a cada melhoria o normal é que sejam diminuídas os números de leituras lógicas e físicas.

Com esse primeiro passo normalmente já conseguimos obter dados para melhorias da ordem de no minimo 50% na maioria dos ambientes, pois normalmente os índices são criados quando o problema estão muito graves, mas depois eles nunca mais são revisados para verificar se devem ser modificados ou mesmo criados novos índices.

É importante a revisão dos índices pelo menos mensalmente porque  qualquer modificação de sistema pode tornar alguns índices obsoletos e pedir novos índices, assim como a revisão das estatistas, pois essas são essenciais para o SQL Server utilizar o indice correto.

 Conclusão

Otimizar SQL Server não deve ser uma tarefa a ser realizada uma unica vez deve ser repetida de tempos em tempos com critério e usando conhecimentos já consagrados pela comunidade técnica.
